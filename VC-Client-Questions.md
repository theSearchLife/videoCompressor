# Client Questions

Please answer these in plain English. Short answers are fine.

The goal of this phase is simple: I want to test the tool against the kind of videos you really have, then use your feedback to lock the requirements before I build the rest.

## 1. Your Input Videos

1. What kind of videos are these?
   Examples: phone videos, camera footage, screen recordings, drone footage, downloaded videos, or a mix.

2. Roughly how large are the files?
   Example: "usually 200 MB to 2 GB each" or "sometimes 10 GB+".

3. Roughly how many videos would you compress in one run?
   Example: 5, 20, 100, or more.

4. Can you share 2-3 real sample videos?
   This is the most helpful part. Testing your real input files will give much better results than testing only a generic demo clip.

5. Are your videos all in one folder, or spread across subfolders?

## 2. Quality Expectations

6. Where will you usually watch the compressed videos?
   Example: phone, laptop, TV, browser, VLC, Plex, or something else.

7. Which matters more to you?
   Smaller files, faster compression, or a balance between both.

8. What is the minimum acceptable quality?
   Example: "must still look good on a TV", "just needs to be clearly watchable", or "I want to compare samples first".

## 3. Special Cases

9. Do any of your videos use HDR?
   If you are not sure, that is fine.

10. Is the audio important?
    Example: speech-heavy videos, music, or audio that does not matter much.

11. Do any videos have subtitles or more than one audio track?

## 4. Input And Output Location

12. Where will the input videos come from?
    Example: "I will point the tool at one folder on my drive" or "I want to drop videos into a folder and run it there".

13. Where should the compressed videos go?
    Choose whichever is easier for you:
    - Next to the original files
    - In a separate output folder

14. Should the original files always be kept?
    The current plan is yes: the tool never edits or deletes the originals.

## 5. How You Will Run It

15. Will you run this on Mac, Linux, or both?

16. Do you also need Windows support?

17. Will you run it manually, or should it be easy to schedule later?

## What I Need From You For Phase 0

If possible, please send:

1. Answers to the questions above
2. Two or three sample videos
3. Confirmation of where input files live and where output files should go

## What Happens Next

After that:

1. I run the assessment tool on your sample videos
2. You review the generated output files and the comparison report
3. You tell me which results look good enough
4. I lock the final settings and complete the remaining phases
